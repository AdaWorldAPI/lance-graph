# User / Agent / Topic Ripple Model

## Why this note exists

The architecture has mostly been discussed in terms of anatomy, representation layers, resonance, and bus execution.

This note adds another necessary axis:

cognition is also a relation among user, agent, topic, and angle.

Without that, the system risks becoming structurally rich but interpersonally flat.

## Main idea

Every meaningful thought event can be seen as the interaction of four poles:

- the user
- the agent
- the topic
- the angle or perspective

These do not merely decorate the same content.
They change the traversal, the hypotheses, the synthesis, and the reward landscape.

## The graph view

In graph terms:

- user = a dynamic node cluster with preferences, episodes, and active tensions
- agent = a dynamic node cluster with mode, policy, and current stance
- topic = an active region of graph structure and family space
- angle = a selected projection or predicate mask over the topic

Thought then becomes not just node retrieval, but movement through perspective-conditioned graph topology.

## The ripple view

In ripple terms:

- user state perturbs the field
- agent stance perturbs the field
- topic families activate regions of the field
- angles act like selective wavefront filters
- hypotheses form as interference ridges
- synthesis is the temporary stabilization of overlapping and conflicting pressures

## The spine view

In trajectory terms:

- episodes form directed causal arcs
- graph revisions accumulate as support or contradiction
- persona shifts alter traversal bias
- successful synthesis forms a durable spine

So cognition can be tracked in two complementary ways:

- as a ripple field of active pressures
- as a spine trajectory of causal stabilization

## Shared gestalt

A useful object for the architecture is a shared gestalt between user and agent.

This should preserve:

- overlap
- mismatch
- unresolved tension
- active synthesis pressure

A system that only stores overlap becomes flatter than the conversation.
A system that only stores conflict becomes brittle.
A good shared gestalt stores both.

## Mirror-style modeling

A mirror-style loop can be implemented without anthropomorphic excess:

1. estimate likely user frame
2. estimate current agent frame
3. compute overlap and tension
4. choose whether to:
   - align
   - clarify
   - deepen
   - contrast
   - synthesize

This can guide both traversal and response policy.

## AriGraph + persona + episodic memory

These should be treated as one linked stack:

- AriGraph = explicit structural memory and inference surface
- persona modeling = viewpoint and policy surface
- episodic memory = causal trajectory over time

Together, they enable not just remembering facts, but remembering how perspectives moved.

## Semantic kernel role

A semantic kernel can be framed as the compact mediation layer between:

- ripple field state
- perspective state
- explicit traversal
- synthesis output

This makes it a useful compression and coordination surface rather than a buzzword bucket.

## Reinforcement role

A reinforcement signal should not only reward token success.
It can reward:

- trajectory coherence
- contradiction handling
- stable synthesis
- appropriate novelty
- perspective alignment without collapse into mimicry

This creates a more structural learning target for `p64 + Blumenstrauß` traversal and collapse policy.

## Immediate implementation implications

1. Add first-class user and agent frames.
2. Add perspective and angle objects that can survive before textualization.
3. Add trajectory arcs to episodic memory.
4. Let `ThoughtStruct` include user-agent overlap and tension fields.
5. Benchmark reinforcement over traversal quality, not only output string quality.
